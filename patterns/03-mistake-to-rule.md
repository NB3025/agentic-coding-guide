# Pattern 3: Mistake → Rule

## 문제

에이전트가 같은 실수를 반복한다. 매번 지적해도 다음 세션에서 또 한다.

## 해결

**반복된 실수를 명시적 규칙으로 승격**시킨다. 학습 → 규칙 파이프라인.

## 메커니즘

```
실수 발생 → learnings.md에 기록 → 3회 반복 → boundaries.md에 영구 추가
```

## 학습 기록 형식

```markdown
## 2026-04-14 — auth 리팩토링

### 실패한 접근법
- bcrypt 직접 import해서 사용
  - Why: 프로젝트에서 argon2 통일 사용 중이었음
  - How to apply: 기존 코드의 해싱 방식 먼저 확인

### 발견한 패턴
- 기존 유틸 함수 존재 여부 먼저 검색
  - Why: 중복 구현 방지
  - How to apply: 새 함수 작성 전 `grep -r` 실행
```

## 승격 기준

- 3회 이상 반복 → `boundaries.md`에 추가
- 60일 이상 경과 + 미참조 → 삭제 (학습은 영원하지 않다)
- 50건 초과 시 오래된 항목부터 삭제

## 핵심

규칙은 프로젝트 시작 전에 완성되는 게 아니다. **에이전트와 함께 진화한다.**
