# Pattern 12: Security & Performance Guard

## 문제

기능은 되는데 보안 구멍이 있거나, 성능이 끔찍하다. 에이전트는 "동작하면 끝"이라 생각한다.

## 해결

**완료 기준에 보안·성능 검증을 포함**한다.

## 보안 체크

```markdown
- [ ] 사용자 입력 검증 (SQL injection, XSS 등)
- [ ] 시크릿이 코드에 없는가?
- [ ] 인증/인가가 적용되었는가?
- [ ] 에러 메시지에 내부 정보가 노출되지 않는가?
```

## 성능 체크

```markdown
- [ ] N+1 쿼리가 없는가?
- [ ] 불필요한 데이터 로딩이 없는가?
- [ ] 루프 내 API/DB 호출이 없는가?
- [ ] 큰 데이터셋에서도 동작하는가?
```

## 성능 가드레일

```markdown
### 🚫 Never
- N+1 쿼리
- 루프 내 API 호출
- 불필요한 전체 테이블 스캔
- unbounded 쿼리 (LIMIT 없는 SELECT)
```

## Pattern 6 (Self-Review)와 연동

이 항목들은 Self-Review 체크리스트에 포함시켜 매 태스크마다 자동 검증한다.
