# Pattern 9: Doc Sync

## 문제

코드는 바뀌었는데 문서는 그대로다. API가 바뀌었는데 README는 옛날 예시.

## 해결

코드 변경 시 **관련 문서도 함께 업데이트**를 강제한다.

## 체크 포인트

- API 엔드포인트 변경 → API 문서 업데이트
- 설정 옵션 추가/변경 → README 또는 설정 가이드 업데이트
- 새 의존성 추가 → 설치 가이드 업데이트
- 복잡한 로직 → 인라인 주석 추가

## 안티패턴

- ❌ "나중에 문서 정리할게" → 안 함
- ❌ 코드와 문서를 별도 커밋 → 중간에 불일치 상태 생김

## 규칙

코드 변경과 문서 변경은 **같은 커밋**에 포함한다.
