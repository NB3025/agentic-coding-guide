# 경계와 가드레일

## 3단계 경계 (#4 금지 사항 + #12 보안·성능)

### ✅ Always
- 테스트 작성 후 커밋
- 입력 검증 (사용자 입력, API 응답)
- 환경변수로 시크릿 관리
- 에러 핸들링 포함

### ⚠️ Ask first
- 기존 아키텍처 변경
- 새 외부 의존성 추가
- DB 스키마 변경
- 기존 API 인터페이스 변경

### 🚫 Never
- 시크릿을 코드에 하드코딩
- 프로덕션 DB에 직접 접근
- 테스트 없이 커밋
- 기존 테스트 삭제 (수정은 OK)
- `any` 타입 남용 (TypeScript)
- console.log 디버깅 코드 커밋

## 성능 가드레일
- N+1 쿼리 금지
- 불필요한 데이터 로딩 금지
- 루프 내 API 호출 금지

## 이 파일은 진화한다 (#3 반복 실수 → 규칙 추가)
프로젝트 초기에는 보편적 규칙만. 에이전트의 반복 실수가 관찰되면 여기에 추가.

## 디버깅 (#7 Git Bisect)
- 문제 발생 시 `git bisect`로 원인 커밋을 이진 탐색
- 자동화: `git bisect run <test-command>`
- 전제: 커밋 단위가 작고 각 커밋이 빌드 가능해야 함
